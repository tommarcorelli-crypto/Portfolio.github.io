Réalisation technique実装

01

Préparation des machines virtuelles

Avant toute installation de rôle, deux actions sont indispensables : nommer le serveur de manière explicite et lui attribuer une adresse IP fixe. Un contrôleur de domaine doit toujours être joignable à la même adresse.

Configuration réseau — SRV-AD-01
Carte VMnet2192.168.2.10 / 255.255.255.0
Passerelle— (réseau isolé)
DNS préféré (provisoire)127.0.0.1 (loopback)
Carte NATAccès Internet (mises à jour)
💡

La convention SRV-AD-01 suit le modèle TYPE-RÔLE-NUMÉRO — pratique courante pour identifier rapidement une machine sur le réseau.

02

Installation du rôle AD DS

Le rôle AD DS transforme le serveur en contrôleur de domaine. L'installation se déroule en deux phases via le Gestionnaire de serveur.

1
Ajout du rôle

Gestionnaire de serveur → Gérer → Ajouter des rôles → Services de domaine Active Directory. Les dépendances (module PowerShell AD, outils d'administration) sont installées automatiquement.

2
Promotion en contrôleur de domaine

Via la notification post-installation → Promouvoir ce serveur en contrôleur de domaine. Choix : Créer une nouvelle forêt avec le nom racine sisr.local. Un mot de passe DSRM est défini pour la récupération d'urgence.

ℹ️

Le suffixe .local désigne un domaine privé non résolvable sur Internet. Le rôle DNS est installé automatiquement avec AD DS — les postes clients en ont besoin pour localiser le contrôleur via des enregistrements SRV.

03

Création de la structure organisationnelle

Les objets sont organisés dans des Unités d'Organisation (OU) — des conteneurs logiques qui permettent d'appliquer des GPO ciblées.

📁

OU Utilisateurs-SISR

Créée directement sous sisr.local via la console Utilisateurs et ordinateurs AD.

👤

jean.dupont

Format prénom.nom@sisr.local — convention standard lisible et unique en entreprise.

👤

Marie Martine

Même convention. Option forçant le changement de mot de passe à la première connexion.

👥

GRP-Employes

Type Sécurité, étendue Domaine local. Jean et Marie ajoutés via l'onglet Membres.

04

Configuration de la GPO

Une GPO est un ensemble de paramètres qui s'appliquent automatiquement aux utilisateurs ou ordinateurs d'une OU, sans intervention manuelle sur chaque poste.

Liaison GPO — Console GPMC
sisr.local→ Utilisateurs-SISR→ GPO-Employes (liée)
Paramètre configuré dans l'éditeur GPO
CheminConfig. utilisateur → Modèles d'administration → Panneau de configuration
ParamètreInterdire l'accès au Panneau de configuration et à l'app Paramètres
État✅ Activé
Filtre de sécuritéUtilisateurs authentifiés (ou GRP-Employes)
💡

La GPO s'applique à la prochaine ouverture de session, ou immédiatement après gpupdate /force.

05

Configuration et jonction du poste client

CLI-WIN-01 est renommée et configurée avec une IP fixe dans le même sous-réseau. Point critique : le DNS préféré doit impérativement pointer vers 192.168.2.10 pour localiser sisr.local.

Configuration réseau — CLI-WIN-01
Adresse IP192.168.2.20 / 255.255.255.0
DNS préféré192.168.2.10 (SRV-AD-01) ⚠️
Jonction domaineParamètres → Système → À propos → Renommer ce PC (avancé)
IdentifiantsSISR\Administrateur
✅

Un message « Bienvenue dans le domaine sisr.local » confirme la réussite. Après redémarrage, l'écran de connexion propose les comptes du domaine.

Bilan & Compétences総括

⚠️ Difficulté rencontrée et résolue

ProblèmeImpossible de joindre le domaine depuis CLI-WIN-01 malgré ping et nslookup réussis.
CauseDNS préféré de SRV-AD-01 configuré en 127.0.0.1 — le serveur ne s'enregistrait pas correctement dans son propre DNS avec son IP réseau.
SolutionModification du DNS préféré de 127.0.0.1 vers 192.168.2.10. Jonction réussie immédiatement après correction.
MéthodeAnalyse des messages d'erreur Windows, tests ping + nslookup pour isoler la couche réseau de la couche DNS, puis vérification côté serveur.
